# Exhaustive Color Research for Web/UI Design

## 1. COLOR PSYCHOLOGY (Per Color)

### RED
- **Hex**: `#EF4444` (Tailwind red-500) | **OKLCH**: `oklch(0.637 0.237 25.331)`
- **Emotions triggered**: Urgency, passion, danger, excitement, anger, love, power, hunger
- **Physiological effect**: Increases heart rate, raises blood pressure, stimulates appetite
- **When to use in UI**:
  - Error states and validation messages
  - Destructive actions (delete, remove)
  - Sale/discount badges and CTAs ("Buy Now")
  - Notification badges indicating unread/urgent items
  - Food-related apps (stimulates appetite)
  - Emergency/warning alerts
- **When to avoid**:
  - Large background areas (visually aggressive, causes eye strain)
  - Success confirmations (users associate red with errors)
  - Healthcare apps as primary color (associates with blood/pain)
  - Calm/meditation apps (contradicts the mood)
  - Excessive use creates anxiety and tension
- **Industries/brands**: YouTube, Netflix, Coca-Cola, McDonald's, Target, YouTube, Pinterest, Adobe (in error states), food delivery apps, emergency services, retail/sales
- **Pairs well with**: White (clean, high-contrast), black (dramatic, bold), gray (muted professionalism), light blue (contrast), gold (luxury, premium)
- **Shades**:
  - `red-50` `#FEF2F2` — lightest tint, backgrounds
  - `red-100` `#FEE2E2` — light backgrounds
  - `red-200` `#FECACA` — hover states on light bg
  - `red-300` `#FCA5A5` — borders, subtle accents
  - `red-400` `#F87171` — icons, secondary text
  - `red-500` `#EF4444` — primary red, CTAs
  - `red-600` `#DC2626` — hover on primary
  - `red-700` `#B91C1C` — active/pressed states

## 2. COLOR PALETTE TYPES

### MONOCHROMATIC
- **Definition**: Variations in lightness and saturation of a single hue
- **When to use**:
  - Minimal, elegant, focused designs
  - When you want cohesion without complexity
  - Dashboards and data-heavy interfaces
  - When brand color is strong and you want it to dominate
  - Photography portfolios (let content be the color)
  - Corporate/professional sites wanting subtle sophistication
- **Pros**: Always harmonious, easy to implement, clean hierarchy through lightness
- **Cons**: Can feel flat or boring without careful value contrast, limits visual interest
- **Example (Blue monochromatic)**:
  ```
  Background:  bg-blue-50    (#EFF6FF)
  Surface:     bg-blue-100   (#DBEAFE)
  Border:      border-blue-200 (#BFDBFE)
  Muted text:  text-blue-400  (#60A5FA)
  Body text:   text-blue-700  (#1D4ED8)
  Heading:     text-blue-900  (#1E3A8A)
  CTA:         bg-blue-600    (#2563EB)
  ```

### ANALOGOUS
- **Definition**: 2-3 colors adjacent on the color wheel (within 60-90 degrees)
- **When to use**:
  - Nature/organic themes (naturally harmonious)
  - When you want color variety without conflict
  - Gradient designs
  - Seasonal themes (autumn: orange + red + yellow)
  - Landing pages needing visual richness
  - Wellness, health, and environment brands
- **Pros**: Naturally pleasing, easy to balance, rich without being chaotic
- **Cons**: Needs a clear dominant color, can lack contrast for CTAs
- **Example (Ocean analogous — blue + teal + green)**:
  ```
  Primary:     bg-blue-600    (#2563EB)
  Secondary:   bg-teal-500    (#14B8A6)
  Accent:      bg-green-500   (#22C55E)
  Background:  bg-blue-50     (#EFF6FF)
  ```
- **Example (Warm analogous — red + orange + yellow)**:
  ```
  Primary:     bg-orange-500  (#F97316)
  Secondary:   bg-red-500     (#EF4444)
  Accent:      bg-yellow-500  (#EAB308)
  ```

### COMPLEMENTARY
- **Definition**: Two colors opposite on the color wheel (180 degrees apart)
- **When to use**:
  - When you need maximum contrast and visual tension
  - CTAs that must pop against the background
  - Sports teams, bold brands
  - Highlighting important information
  - Call-to-action heavy pages
  - When one color is dominant and the other is accent only
- **Pros**: Maximum visual impact, natural tension draws attention, vibrant
- **Cons**: Can be jarring if used equally (50/50), needs careful balance
- **Example pairs**:
  - Blue `#3B82F6` + Orange `#F97316` (most popular in UI)
  - Purple `#A855F7` + Yellow `#EAB308`
  - Red `#EF4444` + Green `#22C55E`
  - Teal `#14B8A6` + Rose `#F43F5E`
- **Usage pattern**: 70% dominant + 25% neutral + 5% complement as accent

### SPLIT-COMPLEMENTARY
- **Definition**: One base color + two colors adjacent to its complement
- **When to use**:
  - When you want complementary energy but softer tension
  - More nuanced than straight complementary
  - Designs needing three distinct colors
  - When complementary feels too harsh
  - Creative agencies, design portfolios
- **Pros**: Strong visual contrast but more nuanced, harder to mess up than complementary
- **Cons**: More colors to manage, needs clear hierarchy
- **Example (Blue base)**:
  ```
  Base:              blue-600    (#2563EB)
  Split-complement 1: orange-400 (#FB923C) — yellow-orange side
  Split-complement 2: rose-500   (#F43F5E) — red-orange side
  ```
- **Example (Green base)**:
  ```
  Base:              green-600   (#16A34A)
  Split-complement 1: pink-400   (#F472B6) — red side
  Split-complement 2: violet-400 (#A78BFA) — purple side
  ```

### TRIADIC
- **Definition**: Three colors equally spaced on the color wheel (120 degrees apart)
- **When to use**:
  - Playful, vibrant, youthful designs
  - Children's products and educational platforms
  - Festival/event websites
  - When you need three distinct but harmonious colors
  - Game interfaces
- **Pros**: Vibrant, balanced, offers variety
- **Cons**: Can feel chaotic if not carefully balanced, needs one dominant
- **Example (Primary triadic)**:
  ```
  Red:    #EF4444 (primary)
  Blue:   #3B82F6 (secondary)
  Yellow: #EAB308 (accent)
  ```
- **Example (Modern triadic)**:
  ```
  Violet:  #8B5CF6 (primary)
  Emerald: #10B981 (secondary)
  ```

- **Definition**: Four colors equally spaced (90 degrees apart), forming a square on the color wheel
- **When to use**:
  - Complex, information-rich designs
  - Multi-section landing pages (each section gets a color)
  - Dashboards with distinct modules
  - Festival/carnival themes
  - When you have four distinct content areas
- **Pros**: Maximum variety, very vibrant, can tell complex visual stories
- **Cons**: Hardest to balance, easily chaotic, needs strong hierarchy
- **Example**:
  ```

  Yellow:  #EAB308
  Teal:    #14B8A6
  Purple:  #A855F7
  ```

### ACHROMATIC
- **Definition**: No hue — only grayscale (black, white, gray)
- **When to use**:
  - Typography-focused designs (editorial, news)
  - Legal/institutional sites
  - High-art/photography portfolios
  - When content IS the color (photos, videos)
  - Minimalist luxury
  - Wireframes and prototypes
- **Pros**: Timeless, sophisticated, lets content shine, no color clash

- **Definition**: Mostly neutral (gray, beige, slate) with ONE bold accent color
- **When to use**:

## 4. OKLCH COLOR SPACE

### What is OKLCH?

OKLCH (Oklab Lightness Chroma Hue) is a **perceptually uniform** color space, meaning:
- Equal numerical changes produce **equal visual changes**
- Lightness of 0.5 always looks like "50% light" regardless of hue
- Saturation (chroma) changes are visually consistent across hues

**Three components**:
- **L (Lightness)**: 0 = black, 1 = white. Perceptually linear.
- **C (Chroma)**: 0 = gray, ~0.37 = maximum saturation. Like "saturation" but perceptually uniform.
- **H (Hue)**: 0-360 degrees. Red=25, Yellow=95, Green=145, Cyan=195, Blue=265, Purple=305

### Why OKLCH is Better Than HSL

| Problem with HSL | OKLCH Solution |
|---|---|
| `hsl(0, 100%, 50%)` (red) looks darker than `hsl(120, 100%, 50%)` (green) at same lightness | L=0.7 in OKLCH looks equally bright for ALL hues |
| Creating shades by varying L in HSL produces inconsistent perceived brightness | Varying L in OKLCH produces perceptually even steps |
| Blue at `L=50%` is nearly unreadable, yellow at `L=50%` is fine | OKLCH compensates — same L means same readability |
| Mixing colors in HSL produces muddy midpoints | OKLCH mixing produces clean, predictable results |
| Grays in HSL can have subtle color casts | OKLCH chroma=0 is truly neutral gray |

### CSS Syntax

```css
/* OKLCH in CSS */
color: oklch(0.7 0.15 180);
/*          L   C    H   */

/* With alpha */
color: oklch(0.7 0.15 180 / 0.5);

/* Functional notation examples */
background: oklch(0.97 0.01 250);    /* very light blue tint */
text:       oklch(0.25 0.05 250);    /* very dark blue */
accent:     oklch(0.65 0.25 250);    /* vibrant blue */
gray:       oklch(0.55 0 0);         /* neutral gray (chroma=0) */
```

### How to Create Palettes in OKLCH

**Method: Vary Lightness, Keep Chroma/Hue Constant**

```css
/* Blue palette generation */
--blue-50:  oklch(0.97 0.01 250);   /* lightest */
--blue-100: oklch(0.93 0.03 250);
--blue-200: oklch(0.87 0.07 250);
--blue-300: oklch(0.78 0.12 250);
--blue-400: oklch(0.68 0.19 250);
--blue-500: oklch(0.62 0.21 250);   /* primary/base */
--blue-600: oklch(0.55 0.22 250);
--blue-700: oklch(0.47 0.20 250);
--blue-800: oklch(0.39 0.15 250);
--blue-900: oklch(0.33 0.10 250);
--blue-950: oklch(0.24 0.06 250);   /* darkest */
```

**Advanced: Chroma Curve**

For more realistic palettes, chroma isn't constant — it peaks at 500 and tapers at extremes:

```css
/* Realistic palette with chroma curve */
--red-50:  oklch(0.97 0.01 25);
--red-100: oklch(0.94 0.03 25);
--red-200: oklch(0.88 0.06 25);
--red-300: oklch(0.81 0.12 25);
--red-400: oklch(0.71 0.19 25);
--red-500: oklch(0.64 0.24 25);    /* peak chroma */
--red-600: oklch(0.58 0.25 25);    /* chroma still high */
--red-700: oklch(0.50 0.21 25);    /* chroma starts dropping */
--red-800: oklch(0.44 0.18 25);
--red-900: oklch(0.40 0.14 25);
--red-950: oklch(0.26 0.09 25);    /* chroma very low */
```

**Key rule**: Very light shades (50-200) and very dark shades (800-950) should have LOW chroma. Mid-tones (400-600) carry the most chroma.

### Color Blindness Considerations

**Types and prevalence**:
- **Protanopia** (1% of males): Cannot see red. Red appears as dark brown/gray.
- **Deuteranopia** (1% of males): Cannot see green. Green appears as beige/brown. Most common.
- **Tritanopia** (0.003%): Cannot see blue. Blue appears as green. Rare.
- **Achromatopsia** (0.003%): Total color blindness. Only sees shades of gray.

**Rules for color-blind-friendly design**:
1. **Never use color alone to convey meaning**. Always add:
   - Text labels ("Error", "Success")
   - Icons (checkmark, X, exclamation)
   - Patterns or shapes
   - Underlines for links (not just color)
2. **Avoid problematic color pairs**:
   - Red + Green (protanopia/deuteranopia)
   - Green + Brown
   - Blue + Purple (tritanopia)
   - Green + Orange
   - Light green + Yellow
3. **Use color-blind-safe palettes**:
   - Blue + Orange (safe for all types)
   - Blue + Yellow (safe)
   - Purple + Green (use with caution, add patterns)
4. **Test with simulation tools** (see below)

### Accessibility Tools

- **WebAIM Contrast Checker** (webaim.org/resources/contrastchecker): Input two hex values, get AA/AAA pass/fail
- **Stark** (getstark.co): Figma/Sketch plugin for contrast checking and color blindness simulation
- **Chrome DevTools**: Rendering panel has vision deficiency emulation
- **Colour Blind Simulator** (color-blindness.com/coblis-color-blindness-simulator): Upload images to see how color-blind users see them
- **Polypane** browser: Built-in accessibility checks
- **axe DevTools**: Browser extension for full accessibility audit
- **Who Can Use** (whocanuse.com): Shows how color combinations look to different vision types

### Dark Mode Color Trends

Key principles for modern dark mode (2024-2026):
1. **Never use pure black** (`#000000`). Use dark gray-blue (`oklch(0.13 0.02 260)`)
2. **Dark mode is not just inverted light mode**. Colors need different chroma/lightness.
3. **Reduce chroma in dark mode**. Vibrant colors on dark backgrounds appear to "glow" and become harder to read.
4. **Use lighter text, not brighter colors**. `oklch(0.85-0.95)` for body text on dark.
5. **Elevation = lighter surfaces**. Cards/modals slightly lighter than background.
6. **Accent colors need lightness adjustment**. A blue that works on white (`oklch(0.62 0.21 260)`) needs to be lighter on dark backgrounds (`oklch(0.70 0.18 260)`).

## QUICK REFERENCE: PALETTE GENERATION FORMULA

To generate a full 11-shade palette from any OKLCH color `oklch(L C H)`:

```javascript
function generatePalette(baseL, baseC, baseH) {
  const steps = {
    50:  { L: 0.97, C: baseC * 0.05 },
    100: { L: 0.94, C: baseC * 0.14 },
    200: { L: 0.87, C: baseC * 0.32 },
    300: { L: 0.78, C: baseC * 0.55 },
    400: { L: 0.72, C: baseC * 0.86 },
    500: { L: baseL, C: baseC },           /* BASE */
    600: { L: baseL * 0.85, C: baseC * 1.0 },
    700: { L: baseL * 0.72, C: baseC * 0.91 },
    800: { L: baseL * 0.60, C: baseC * 0.68 },
    900: { L: baseL * 0.50, C: baseC * 0.50 },
    950: { L: baseL * 0.37, C: baseC * 0.27 },
  };

  return Object.entries(steps).map(([step, { L, C }]) =>
    `--color-${step}: oklch(${L.toFixed(3)} ${C.toFixed(3)} ${baseH});`
  ).join('\n');
}
```
